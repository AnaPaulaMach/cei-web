# Guía visual CEI

## Concepto

La identidad visual del sitio es la de un mini escritorio retro de la Comisión de Estudiantes de Informática. Debe sentirse como un espacio informático universitario, con ventanas, barra de tareas, accesos directos, paneles de sistema, terminales y detalles pixelados.

La referencia es Windows XP / Minecraft / informática retro, pero adaptada a CEI. No se copian marcas, logos ni interfaces exactas.

## Paleta principal

- Azul sistema: `#1d4ed8`
- Azul oscuro: `#0f2a5f`
- Verde terminal: `#22c55e`
- Gris ventana: `#e5e7eb`
- Fondo laboratorio: `#102a43`
- Amarillo alerta: `#fde047`
- Blanco papel: `#ffffff`
- Negro terminal: `#07111f`

La paleta debe sentirse como un laboratorio de informática retro: más oscura y contenida que Windows XP puro, con azul como estructura, verde terminal para estados y amarillo solo para llamados importantes.

## Tipografías

- Texto largo: sans legible del sistema.
- Títulos, badges y pequeños labels: monospace o pixel-like.
- Evitar usar fuente pixel para párrafos largos o formularios.

## Componentes

- Ventanas: bordes marcados, barra superior azul, botones pequeños tipo minimizar/maximizar/cerrar.
- Botones: relieve clásico, foco visible, estados hover claros.
- Tarjetas: pueden parecer archivos, accesos directos o paneles de sistema.
- Iconos: usar estilo pixel/desktop, simples y reconocibles.
- Formularios: priorizar legibilidad, labels claros y buen espaciado.
- Barra de tareas: puede aparecer como elemento persistente o decorativo, sin tapar contenido.

## Tono visual

Joven, informático, cercano y universitario. La estética puede tener humor visual, pero la información institucional debe seguir siendo seria.

## Evitar

- Que la estética retro vuelva difícil leer o completar formularios.
- Usar demasiadas animaciones al mismo tiempo.
- Copiar logos, textos o interfaces exactas de marcas registradas.
- Depender solo de color para transmitir estados.
- Ocultar links importantes solo en el footer.
